GigaPet KnockOff

This semester, I am taking CIT384/L alongside the rest of my courses. CIT384 is a Web Developing and Hosting class taught by Professor Marin. In the class we explore internet infrastructure and networking technologies. Topics we cover include: system and software architectures for web applications and e-business systems, principles of website design, advances in web-engineering technologies, principles of web-based transaction processing, XML and the associated technologies, web service technology, security and privacy issues, and so on. In this week's blog, I will discuss our second individual project for CIT384 where we add actions and functions to our GigaPet.

In last week's lecture, we were assigned Project 2 where we were to make our own GigaPet in the span of one week. This is our first assignment/lecture in this class in which we utilize JavaScript.

In Blog 0, I briefly discussed the basics of HTML and how it can be utilized to work closely together with CSS and JavaScript to create an ideal, well put-together webpage.

From a basic point of view, the structure of a webpage relies heavily on HTML. On the other hand, CSS controls a page's layout and design, while JavaScript works with how it should behave. Together, they are able to create a cohesive and aesthetic webpage.

Steps I Took

For this assignment, I had a total of three files: HTML for the actual webpage, CSS for the layout of the HTML webpage, and JavaScript which contained the functions that regarded any interaction done on the page.

For out GigaPet, we were free to choose any icon. To add you icon, you can add your own image or make other media that include more animations. I added a simple image with a clear background to my HTML page by doing the following: 

    <section class="pet-image-container">
         <!-- Replace pet image with your own pet image -->
         <img class="pet-image" src="https://cdn.glitch.com/734d6424-025d-4912-805b-bc1ef994f0a1%2Fkirby.png?v=1604621243925">
    </section>

JavaScript Functions

Moving on to JavaScript, create and utilize functions in order to add actions/interactions to the webpage.

After choosing an icon, create buttons to interact with. To do so, do the following: 

$(function() { // Makes sure that your function is called once all the DOM elements of the page are ready to be used.
    
    // Called function to update the name, happiness, and weight of our pet in our HTML
    checkAndUpdatePetInfoInHtml();
  
    // When each button is clicked, it will "call" function for that button (functions are below)
    $('.treat-button').click(clickedTreatButton);
    $('.play-button').click(clickedPlayButton);
    $('.exercise-button').click(clickedExerciseButton);
    $('.hydration-button').click(clickedHydrationButton); 
})

Each button can be customized to your liking. For this particular project, I had four basic buttons that had a basic emulation of standard Tomodatchi actions. After establishing buttons, add respective functions and arguments to each.

For this particular project, the goal was to create a basic counter that increased and decreased after interacting with the buttons. And now that the buttons have been established, declare the var statements in order to initialize them in later functions. 

var pet_info = {
      name:"Kirby", 
      weight:"23", 
      happiness:"98", 
      hydration:"49"};

Values can be preset as seen above. To do basic increments, the following is a possible option: 

function clickedTreatButton() {
      // Increase pet happiness
        pet_info.happiness++;
      // Increase pet weight
        pet_info.weight++;
      checkAndUpdatePetInfoInHtml();
      alert("Yum!!");
}

Note: If your function, as seen above, is not reading as an integer, simply remove the quotation marks.

From here, more customizations can be done to your GigaPet. Animations, music, and other conditionals can be added for further effects. I, for one, added small audio clips of Kirby to match my buttons. To accomplish so, I included a "var Audio" (where the audio can be found) followed by "audio.play();".
